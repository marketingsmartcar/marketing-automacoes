## SOBRE ESTE PROJETO

Este repositório contém o sistema de automação de marketing do **Grupo BR Pneus / Peg Pneus**, composto por agentes especializados e skills reutilizáveis. O sistema atende **duas marcas distintas** — BR Pneus & Oficina e Peg Pneus Atacarejo — cada uma com identidade visual, tom de voz e público próprios. Todo conteúdo gerado deve identificar claramente para qual marca se destina antes de aplicar qualquer diretriz.

## TOM DE VOZ E DIRETRIZES DE COMUNICAÇÃO

### Como a marca fala
- **Popular, direto e confiável** — nunca elitista, nunca técnico demais
- Linguagem acessível para público classe B/C
- Apela a três valores centrais: **economia**, **segurança** e **praticidade**
- Usa verbos de ação, frases curtas e CTAs objetivos
- Pode usar humor leve, mas nunca vulgar ou polêmico
- Empático: entende que o cliente tem receio de oficinas

### O que a marca NUNCA faz
- Usar jargão técnico sem explicar
- Fazer promessas que não pode cumprir
- Adotar tom agressivo ou de pressão excessiva
- Criticar concorrentes pelo nome
- Usar linguagem elitista ou de luxo

### Pilares de conteúdo
1. **Educação** — ensinar o cliente a cuidar do carro
2. **Economia** — mostrar que cuidar previne gastos maiores
3. **Segurança** — conectar manutenção com proteção da família
4. **Promoção** — ofertas diretas com urgência e clareza
5. **Prova social** — depoimentos reais, avaliações, antes/depois

## REGRAS UNIVERSAIS — TODOS OS AGENTES DEVEM SEGUIR

### Regras Gerais (ambas as marcas)
1. **Identificar a marca** no início de qualquer tarefa — BR Pneus & Oficina ou Peg Pneus Atacarejo — e aplicar as diretrizes da marca correta
2. **CTA obrigatório:** todo conteúdo deve ter uma chamada para ação clara (ligar, agendar, mandar mensagem, visitar)
3. **Preços:** nunca inventar valores — usar expressões como "melhor preço", "preço especial", "consulte na loja"
4. **Promoções:** só criar quando solicitado com prazo e condições definidos
5. **Informações técnicas:** sempre simplificar para o nível do público-alvo
6. **Português do Brasil:** todo conteúdo em pt-BR, gramática correta, sem anglicismos desnecessários
7. **Revisão de marca:** antes de finalizar qualquer peça, verificar alinhamento com tom de voz e identidade visual da marca correta
8. **Output:** salvar os arquivos gerados na pasta `/output/` na subpasta correspondente ao tipo de conteúdo
9. **Imagens reais obrigatórias:** sempre usar fotos reais de `assets/imagens/` (produtos, veículos, fundos) — nunca gerar, ilustrar ou usar SVG como substituto de foto de produto

### Regras Específicas — BR Pneus & Oficina
1. **Nome da marca:** sempre "BR Pneus & Oficina" — nunca abreviar para "BR Pneus", "BRP" ou qualquer outra forma
2. **Taglines:** "Muito mais que pneus" (principal, encerramento de posts) | "Liberdade para Rodar" (institucional e de marca)
3. **Garantia:** sempre "BR Total Plus" (com P maiúsculo) — nunca apenas "BR Total"
4. **Unidades:** ao citar cidade, verificar se há unidade ativa (consultar lista na seção A EMPRESA). Araraquara tem 2 lojas.
5. **Logo obrigatória:** sempre usar `assets/imagens/logos-marcas/brpneus-logo-borda.png` — nunca substituir por texto CSS ou logo gerada

### Regras Específicas — Peg Pneus Atacarejo
1. **Nome oficial:** sempre "Peg Pneus Atacarejo" — nunca apenas "Peg Pneus" em documentos formais, nunca "PEGI"
2. **Nunca usar "único" ou "maior"** — não afirmar que é o único ou maior atacarejo de pneus do Brasil; usar "primeiro" (pioneiro) quando quiser destacar o posicionamento de marca
2. **Tagline:** "O primeiro atacarejo de pneus do Brasil!" — usar para posicionamento de marca
3. **Logo em artes escuras:** usar `assets/imagens/logos-marcas/Logotipo-PEGPNEUS-Aplicações-Fundos_Prancheta 1 cópia 4.png` (branca total)
4. **Logo em artes claras:** usar `assets/imagens/logos-marcas/Logotipo-PEGPNEUS-Aplicações-Fundos_Prancheta 1 cópia 2.png` (colorida verde + preto)
5. **Nunca misturar** cores ou logos da Peg Pneus com a BR Pneus & Oficina
6. **Cor verde `#44C52A`** é exclusiva da Peg Pneus — nunca usar o amarelo `#F5A623` da BR Pneus em peças Peg Pneus
7. **Unidades:** Araraquara e Sorocaba — verificar a unidade correta ao citar endereço ou telefone

## ORQUESTRADOR — Como os Agentes Trabalham Juntos

Quando um pedido é feito, o arquivo [agents/orquestrador.md](agents/orquestrador.md) define qual **FLUXO de agentes** será ativado automaticamente.

### Fluxos Disponíveis

| # | Fluxo | Cadeia Principal |
|---|-------|-----------------|
| 1 | **Post / Arte para Redes Sociais** | SEO → Content → Social Media → Brand Guardian → Creative Designer |
| 2 | **Campanha Completa** | Social Media → SEO → Content → Ads → CRM → Telemarketing → Brand Guardian → Creative → Analytics → Franchise |
| 3 | **Conteúdo de Serviço** | Content → SEO → Social Media → Video → Brand Guardian → Creative |
| 4 | **Material para Unidade Local** | Franchise → Social Media/Content → Brand Guardian → Creative |
| 5 | **Reativação / CRM** | CRM → Telemarketing → Brand Guardian → Creative |
| 6 | **Relatório / Análise** | Analytics → (agente de correção, se necessário) |

### Regra de Ouro
- **Brand Guardian revisa TUDO** antes de qualquer peça ir ao ar
- **Creative Designer é SEMPRE o último** quando arte visual é necessária
- O Orquestrador escolhe o fluxo **automaticamente** baseado no pedido
- Pedidos simples ("só a copy", "só a arte") acionam o agente específico diretamente — sem rodar o fluxo completo

## Fontes de Imagem para Criativos

O Creative Designer aceita imagens de 5 fontes via o campo `"imagem"` no JSON:

| Prefixo | Fonte | Exemplo | Custo |
|---------|-------|---------|-------|
| `local:` | Foto própria em `assets/imagens/` | `local:produtos/pneu-aro14.png` | Grátis |
| `svg:` | Biblioteca SVG inline | `svg:pneu`, `svg:freio`, `svg:carro` | Grátis |
| `emoji:` | Emoji Unicode | `emoji:🛞,120` | Grátis |
| `https://` | URL de banco de imagens | `https://...jpg` | Grátis |
| `gemini:` | Geração por IA | `gemini:car tire close up` | API Key |

**SVGs disponíveis:** `pneu` `chave` `velocimetro` `escudo` `freio` `oleo` `carro` `caminhao` `moto` `check` `estrela` `whatsapp` `telefone` `pin` `seta`

```bash
# Listar tudo disponível
node tools/image-resolver.js --list

# Adicionar fotos próprias
# Copie para: assets/imagens/produtos/, assets/imagens/fachadas/, etc.
```

**Pasta de imagens:** `assets/imagens/` — veja `assets/README.md` para convenções de nomeação.

## Como Ver e Baixar as Artes

### Visualizar no VSCode (Live Preview)
1. O Creative Designer gera o HTML e informa o caminho
2. Abra o arquivo no VSCode → clique com botão direito → **"Show Preview"**
3. O preview aparece no painel lateral direito (atualiza ao vivo)
4. Atalho: `Ctrl+Shift+V` com o arquivo aberto

### Baixar como PNG
```bash
# Uma arte específica
node tools/export-html-to-png.js output/criativos/[arquivo].html

# Todas as artes de uma vez
node tools/export-html-to-png.js --all
```
O PNG é salvo na mesma pasta com o mesmo nome. Resolução 2× (Retina).

Requer Puppeteer instalado: `npm install puppeteer`

## MONITORAMENTO DE ADS

Sistema para acompanhar em tempo real todas as contas de anúncios da rede.

### Contas Monitoradas
- **Meta Ads:** 8 contas (BR Pneus: Ibitinga, Maringá, Americana, Jaú, São Carlos, Araraquara + Peg Pneus: Sorocaba, Araraquara)
- **Google Ads:** 7 contas (BR Pneus: Americana, Araraquara, Jaú, Maringá, São Carlos + Peg Pneus: Araraquara, Sorocaba)

### Comandos
```bash
npm run ads                # Relatório completo (Meta + Google)
npm run ads:meta           # Apenas Meta Ads
npm run ads:google         # Apenas Google Ads
npm run ads:recarregar     # Fluxo interativo de recarga
```

### Regra Crítica de Recarga Meta Ads
> ⚠️ **BR PNEUS ARARAQUARA** usa **Pix nos Fundos** (não no Saldo).

> Esta distinção NUNCA deve ser ignorada nas instruções de recarga.
